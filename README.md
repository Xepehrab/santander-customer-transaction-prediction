# Santander Customer Transaction Prediction

A solution for the [Santander Customer Transaction Prediction](https://www.kaggle.com/c/santander-customer-transaction-prediction) Kaggle competition.

Predict whether a customer will make a transaction in the future using anonymized numeric features.

## Project structure

```
.
├── prepare.py          # Data preprocessing and feature engineering
├── main.py             # LightGBM training and submission
├── train.csv           # Training data (not in git)
├── test.csv            # Test data (not in git)
├── sample_submission.csv
├── train_prepared.feather   # Generated by prepare.py
├── test_prepared.feather    # Generated by prepare.py
└── submission.csv           # Generated by main.py
```

## Setup

1. Download the dataset from Kaggle and place `train.csv` and `test.csv` in the project folder.

2. Install dependencies:

```bash
pip install pandas numpy scikit-learn lightgbm pyarrow
```

## Usage

### Step 1 — Prepare data

```bash
python prepare.py
```

This script:

- Identifies **real** vs **synthetic** rows in the test set (Kaggle trick: real rows have at least one unique value in the test set)
- Builds **frequency-encoded** features (`var_0_freq`, `var_1_freq`, …) from train + real test data
- Sets frequency to `1` for synthetic test rows
- Saves `train_prepared.feather` and `test_prepared.feather`

### Step 2 — Train and predict

```bash
python main.py
```

This script:

- Loads prepared data (runs `prepare.py` automatically if feather files are missing)
- Trains **LightGBM** with 5-fold stratified cross-validation
- Prints fold AUC and overall out-of-fold AUC
- Saves `submission.csv` for Kaggle upload

## Approach

| Step | Method |
|------|--------|
| Feature engineering | Frequency encoding on 200 numeric features |
| Test set handling | Separate real/synthetic rows before encoding |
| Model | LightGBM binary classifier |
| Validation | 5-fold stratified CV, AUC metric |
| Test predictions | Average of fold models |

## Evaluation metric

**AUC** (Area Under the ROC Curve) — higher is better.
